웹 접근성(a11y) 체크 도구입니다.

$ARGUMENTS 에 파일/컴포넌트 경로가 있으면 해당 범위를 검사합니다.
없으면 전체 컴포넌트를 스캔합니다.

## 검사 항목 (WCAG 2.1 기준)

### 1. 시맨틱 HTML
- [ ] div/span 남용 (button, nav, main, article 등 사용)
- [ ] 헤딩 계층 구조 (h1 → h2 → h3 순서)
- [ ] 랜드마크 영역 (header, main, footer, aside)
- [ ] 리스트 마크업 (ul/ol/li)

### 2. 키보드 접근성
- [ ] 모든 인터랙티브 요소 탭 가능
- [ ] 포커스 표시 visible (outline 제거 금지)
- [ ] 포커스 트랩 (모달에서 필수)
- [ ] 스킵 링크 (Skip to content)

### 3. ARIA
- [ ] role 속성 필요한 곳
- [ ] aria-label, aria-labelledby
- [ ] aria-hidden (장식 요소)
- [ ] aria-live (동적 콘텐츠)
- [ ] aria-expanded, aria-selected (상태)

### 4. 폼 접근성
- [ ] label과 input 연결 (htmlFor)
- [ ] 에러 메시지 연결 (aria-describedby)
- [ ] 필수 필드 표시 (aria-required)
- [ ] 자동완성 속성 (autocomplete)

### 5. 이미지/미디어
- [ ] alt 텍스트 (장식이면 alt="")
- [ ] 비디오 자막
- [ ] 오디오 대체 텍스트

### 6. 색상/대비
- [ ] 색상 대비 비율 (일반 4.5:1, 큰 텍스트 3:1)
- [ ] 색상만으로 정보 전달 금지
- [ ] 포커스 표시 충분히 눈에 띄는지

## 출력 형식

```
## 접근성 검사 리포트

### 점수: [A/AA/AAA 수준 달성 여부]

### 심각 (필수 수정)
| 파일:라인 | 문제 | WCAG 기준 | 수정 방법 |
|----------|------|----------|----------|

### 권장 사항
...

### 코드 수정 예시
수정 전 → 수정 후 비교
```

## 자동 수정
간단한 문제는 "바로 수정할까요?" 제안
- alt 누락
- label 연결
- role 추가

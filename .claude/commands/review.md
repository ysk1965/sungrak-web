React/Next.js 코드 리뷰어입니다.

$ARGUMENTS 에 파일 경로나 컴포넌트명이 있으면 해당 파일을 리뷰합니다.
없으면 최근 변경된 파일들을 리뷰합니다.

## 리뷰 체크리스트

### 1. 코드 품질
- [ ] 컴포넌트가 단일 책임 원칙을 따르는가?
- [ ] 불필요한 re-render가 있는가? (useMemo, useCallback 필요 여부)
- [ ] props drilling이 심한가? (Context나 상태관리 필요 여부)
- [ ] 타입스크립트 타입이 적절한가? (any 남용 없는지)

### 2. React 패턴
- [ ] hooks 규칙을 지키는가? (조건문 안에서 호출 등)
- [ ] useEffect 의존성 배열이 올바른가?
- [ ] key prop이 적절한가? (index 사용 지양)
- [ ] 에러 바운더리가 필요한 곳에 있는가?

### 3. 성능
- [ ] 큰 리스트에 가상화(virtualization) 필요한가?
- [ ] 이미지 최적화 (next/image, lazy loading)
- [ ] 번들 사이즈에 영향주는 import가 있는가?

### 4. 접근성
- [ ] 시맨틱 HTML 사용 (div 남용 없는지)
- [ ] aria 속성 필요한 곳에 있는가?
- [ ] 키보드 네비게이션 가능한가?
- [ ] 색상 대비 충분한가?

### 5. 보안
- [ ] XSS 취약점 (dangerouslySetInnerHTML 등)
- [ ] 민감 정보 노출 없는가?
- [ ] API 호출 시 인증/권한 체크


## 5. 디자인 톤앤매너

### 5.1 전반적 인상

| 키워드 | 설명 |
|--------|------|
| **모던 & 세련됨** | SPA 기반의 부드러운 페이지 전환, 깔끔한 레이아웃 |
| **다크 톤 지향** | 어두운 배경 기반의 무게감 있는 분위기 |
| **이미지 중심** | 텍스트보다 이미지/비주얼 에셋 중심의 콘텐츠 전달 |
| **미니멀 구조** | 복잡하지 않은 단순 명료한 섹션 구분 |
| **영문 혼용** | 메뉴, 섹션 타이틀에 영문 적극 활용 (About, Care, Media 등) |

### 5.2 색상 방향성

```
■ Primary Background : 다크 계열 (블랙/다크그레이)
■ Text Primary       : 화이트/밝은 톤
■ Accent             : 브랜드 컬러 (로고 기반)
■ Card Background    : 이미지 오버레이 방식
■ Footer             : 다크 배경 + 밝은 텍스트
```

**분석:**
- 전통적인 교회 사이트의 밝고 따뜻한 분위기와 차별화
- 다크 모드 중심의 모던하고 격식 있는 느낌
- 이미지 에셋에 컬러 의존도가 높음

### 5.3 타이포그래피 방향

- 한글/영문 혼용 시스템
- 슬로건 및 히어로 텍스트: 이미지화하여 브랜드 서체 적용 추정
- 본문: 시스템 폰트 또는 웹폰트 기반 가독성 우선
- 메뉴: 영문 대문자 또는 타이틀케이스

### 5.4 레이아웃 특성

- **그리드**: 3-column 카드 레이아웃 (메인 콘텐츠)
- **여백**: 넉넉한 패딩으로 호흡감 있는 배치
- **섹션 분리**: 풀-와이드 섹션 단위의 스크롤 기반 구성
- **모바일**: 햄버거 메뉴 토글 + 반응형 대응

- 모바일: 햄버거 아이콘 → 슬라이드/아코디언 메뉴
- 로그인 버튼 우측 고정

### 6.2 히어로 섹션

- 풀-와이드 배경 (다크 or 이미지)
- 중앙 정렬 슬로건 텍스트
- 로고/브랜드 이미지 삽입
- 오디오/볼륨 토글 컨트롤 (배경음악 기능)

### 6.3 콘텐츠 카드

- 이미지 기반 카드 (JPG 에셋)
- 3열 균등 분할
- 호버 인터랙션 추정
- 카드 자체가 링크/버튼 역할

### 6.4 CTA 버튼

- "MORE" 텍스트 기반 심플 CTA
- 미니멀 스타일 (배경 없거나 아웃라인 추정)
- 섹션 하단 중앙 배치

### 6.5 소셜 미디어

- SVG 아이콘 기반
- YouTube / Instagram / Facebook 3종
- 푸터 상단 또는 별도 섹션 배치

## 7. 인터랙션 & 미디어

| 요소 | 설명 |
|------|------|
| SPA 라우팅 | 페이지 전환 시 전체 리로드 없이 부드러운 전환 |
| 볼륨 토글 | 배경 음악/오디오 on/off 컨트롤 |
| LIVE 배지 | Media 메뉴에 실시간 방송 표시 |
| 외부 연동 | Google Forms로 신청서/등록 폼 처리 |
| 소셜 링크 | 새 탭으로 외부 SNS 플랫폼 이동 |

## 8. 강점 분석

1. **차별화된 톤앤매너**: 다크 테마 기반으로 일반 교회 사이트와 확실히 다른 모던한 인상
2. **깔끔한 IA**: 5개 메인 카테고리로 명확한 정보 분류
3. **SPA 경험**: SvelteKit 기반의 빠르고 부드러운 페이지 전환
4. **미디어 통합**: LIVE 기능과 소셜 미디어 적극 연동
5. **비주얼 중심**: 이미지/에셋 중심의 직관적 콘텐츠 전달

---

## 9. 개선 가능 포인트

1. **접근성(a11y)**: 다크 테마에서의 색상 대비 확인 필요
2. **SEO**: SPA 특성상 SSR/SSG 적용 여부에 따른 검색 최적화
3. **콘텐츠 밀도**: 메인 페이지 콘텐츠가 3섹션으로 다소 단순
4. **폼 통합**: Google Forms 외부 연동 → 자체 폼으로 통합 시 UX 개선 가능
5. **다국어**: 메뉴는 영문이나 콘텐츠는 한글 — 해외 교인 대상 다국어 지원 고려

## 10. sungrak-web 프로젝트 참고 포인트

현재 sungrak-web 프로젝트에 적용 가능한 레퍼런스:

| POD Church 요소 | sungrak-web 적용 방안 |
|-----------------|----------------------|
| 다크 테마 기반 | variant별 테마 옵션으로 다크/라이트 제공 |
| 3-column 카드 레이아웃 | 메인 섹션 콘텐츠 카드 패턴 참고 |
| SPA 부드러운 전환 | Next.js App Router의 클라이언트 전환 활용 |
| LIVE 배지 UI | 미디어/예배 실시간 스트리밍 표시 참고 |
| 영문 메뉴 + 한글 콘텐츠 | 메뉴 네이밍 전략 참고 |
| 소셜 미디어 통합 | 푸터/별도 섹션 SNS 링크 패턴 |
| 볼륨 토글 | 배경 미디어 컨트롤 UX 아이디어 |
| 외부 폼 연동 | 자체 폼 vs 외부 서비스 선택 기준 참고 |

## 부록: 경쟁사 비교 포지셔닝

```
              밝은/따뜻한 ←──────────→ 어두운/모던
                  │                        │
  전통적 교회 사이트 ●                      ● POD Church
                  │                        │
              단순/정적 ←──────────→ 동적/인터랙티브
```

POD Church는 **모던 + 다크 + 인터랙티브** 포지셔닝으로, 전통적 교회 웹사이트와 명확히 차별화된 디자인 방향을 취하고 있음.

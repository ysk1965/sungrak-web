## 4. 디자인 시스템 분석

### 4-1. 컬러 팔레트

| 용도 | 색상 | 비고 |
|------|------|------|
| 기본 배경 | `#FFFFFF` | 화이트 베이스 |
| 텍스트 (기본) | `#000000` / `#333333` | 본문 텍스트 |
| 텍스트 (라이트) | `#FFFFFF` | 히어로/배너 오버레이 위 |
| 슬라이더 불릿 (비활성) | `#FFFFFF` opacity 0.5 | 반투명 처리 |
| 슬라이더 불릿 (활성) | `#FFFFFF` opacity 1.0 | 현재 슬라이드 |
| 전체 톤 | 무채색 + 이미지 컬러 의존 | 미니멀한 베이스 |

> **특징**: 사이트 자체 브랜드 컬러가 강하지 않고, 배경 이미지와 화이트 스페이스를 중심으로 분위기를 형성. 과도한 색상 사용을 절제한 모노톤 베이스.

### 4-2. 타이포그래피 스케일

```
디바이스별 기본 폰트 크기:
─────────────────────────
PC (≥1024px)     : 18px
태블릿 (≤767px)   : 15px
모바일 (≤467px)   : 14px

폰트 사이즈 클래스:
─────────────────────────
.font-8xl  : 5rem    → 헤드라인, 비전 타이틀
.font-3xl  : 2rem    → 섹션 타이틀
.font-m    : 1.1rem  → 본문 텍스트
.font-xs   : 0.8rem  → 캡션, 보조 텍스트
```

### 4-3. 컴포넌트 패턴

#### 히어로 슬라이더
- **타입**: 전폭(Full-width) 이미지 슬라이더
- **라이브러리**: Swiper.js
- **전환 효과**: Fade (크로스페이드)
- **속도**: 800ms 전환, 5000ms 간격 자동재생
- **페이지네이션**: 하단 불릿 (bottom: 20px)
- **루프**: 무한 반복

#### 정보 카드 (3열 그리드)
- **구성**: 제목 + 부제 + 시간/장소 정보 + CTA 버튼
- **레이아웃**: 균등 3열 배분
- **CTA**: 이미지 기반 "더보기" 버튼 (hover 시 이미지 교체)
- **미디어 링크**: YouTube 아이콘 포함

#### 공지사항 리스트
- **구성**: 제목 텍스트 + 화살표(→) 아이콘
- **노출 개수**: 최대 5건
- **인터랙션**: 클릭 시 상세 페이지 이동

#### CTA 버튼
- **방식**: PNG 이미지 기반 (SVG 미사용)
- **상태**: 기본(default) / 호버(hover) 2장의 이미지 교체
- **파일**: `2_btn_more.png` / `2_btn_more_on.png`

## 5. 반응형 전략

| 브레이크포인트 | 대상 | 주요 변경 |
|---------------|------|----------|
| ≥1024px | PC/데스크톱 | 기본 레이아웃, 18px 베이스 |
| ≤767px | 태블릿 세로 | 15px 베이스, 메뉴 → 햄버거 |
| ≤467px | 모바일 | 14px 베이스, 단일 열 레이아웃 |

**접근 방식**: 폰트 크기 중심 조절 + 레이아웃 구조 유지. 모바일에서 3열 → 1열 스택 전환.

## 6. 디자인 톤앤매너

### 6-1. 전체 무드

| 키워드 | 설명 |
|--------|------|
| 미니멀 | 장식 요소를 최소화하고 콘텐츠에 집중 |
| 화이트스페이스 | 넉넉한 여백으로 시각적 여유 확보 |
| 이미지 중심 | 대형 사진으로 교회 분위기 전달 |
| 정보 전달형 | 예배 시간, 장소 등 실용 정보 우선 |
| 차분함 | 과도한 애니메이션/색상 없이 절제된 톤 |

### 6-2. 디자인 강점

1. **명확한 정보 계층**: 비전 → 예배 → 공지 → 사역 순으로 중요도별 배치
2. **직관적 네비게이션**: 5개 대메뉴 + 드롭다운 서브메뉴로 빠른 탐색
3. **풀스크린 히어로**: 첫인상에서 교회의 분위기를 시각적으로 전달
4. **3열 카드 레이아웃**: 예배 정보를 한눈에 비교 가능
5. **소셜 미디어 통합**: YouTube, Facebook, Instagram 채널 연동

### 6-3. 개선 가능 포인트

1. **PNG 기반 버튼** → SVG 또는 CSS 기반 버튼으로 전환 시 해상도/성능 개선
2. **컬러 시스템 부재** → 브랜드 포인트 컬러 도입으로 정체성 강화
3. **애니메이션 부족** → 스크롤 기반 마이크로 인터랙션 추가 가능
4. **폰트 시스템** → 웹폰트 활용도 향상 여지 (현재 시스템 폰트 의존)
5. **접근성** → 이미지 alt 텍스트, 키보드 내비게이션 보강 가능

## 7. sungrak-web 프로젝트 적용 시사점

### 7-1. 벤치마킹 포인트

| 요소 | 1516교회 접근 방식 | sungrak-web 적용 방안 |
|------|-------------------|---------------------|
| IA 구조 | ABOUT / WORSHIP / CARE / NEXT / NOTICE | 유사한 5~6개 대분류 메뉴 구조 검토 |
| 히어로 | 풀스크린 Fade 슬라이더 | Framer Motion 기반 히어로 애니메이션 |
| 예배 정보 | 3열 카드 그리드 | Tailwind grid + 카드 컴포넌트 |
| 공지사항 | 5건 리스트 + 화살표 | 최신 공지 리스트 위젯 |
| 반응형 | 3단계 브레이크포인트 | Tailwind 기본 브레이크포인트 활용 |

### 7-2. 차별화 방향

| 항목 | 1516교회 | sungrak-web 차별화 |
|------|---------|-------------------|
| 기술 | HTML/CSS/JS (전통적) | Next.js + TypeScript (모던) |
| 애니메이션 | Swiper.js 슬라이더만 | Framer Motion 풍부한 인터랙션 |
| 디자인 | 이미지 의존 (PNG 버튼) | CSS/SVG 기반 + 디자인 시스템 |
| 컬러 | 무채색 위주 | 브랜드 컬러 시스템 적극 활용 |
| 컴포넌트 | 페이지 단위 | 재사용 가능한 컴포넌트 시스템 |
| 다크모드 | 미지원 | variant별 다크/라이트 지원 |
